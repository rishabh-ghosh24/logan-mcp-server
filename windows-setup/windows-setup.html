Logan MCP Windows Setup

Use this guide to connect Codex App on Windows to the shared Logan MCP server.

Recommended: One-Time Codex Installer

For team Windows machines, use the one-time installer instead of configuring PuTTY or manually entering SSH arguments in Codex.

Package these files together in one folder:

windows-setup/
Double-Click-to-Install.cmd
logan-mcp.ps1
logan.key
windows-setup.html
  1. Double-click Double-Click-to-Install.cmd.
  2. Enter your Logan username in strict firstname.lastname format.
  3. Let the installer close Codex when prompted, or close Codex completely from Task Manager.
  4. Open Codex App normally.
The installer normalizes uppercase input to lowercase. For example, Rishabh.Ghosh becomes rishabh.ghosh.

The generated Codex MCP server name is logan-mcp.

Sharing note: Add the real logan.key file locally, zip the windows-setup folder, and send it through your approved internal channel. Do not commit logan.key to git.

Using Logan MCP In Codex

Open a new Codex chat and type:

connect to logan-mcp server

After Codex confirms the logan-mcp server is connected, continue in natural language. For example, ask it to list log sources, investigate recent errors, or run a Log Analytics query.

What The Installer Does

  • Copies logan.key to a fresh key file under %USERPROFILE%\.logan-mcp\.
  • Writes a pinned SSH known_hosts file for 130.162.53.112.
  • Backs up %USERPROFILE%\.codex\config.toml if it already exists.
  • Appends or replaces only [mcp_servers.logan-mcp].
  • Tests SSH connectivity before writing the Codex config.
  • Asks whether to close running Codex processes so the new config can load.

Private Key Format

The file beside the installer must be named logan.key and must be an OpenSSH private key.

Valid private key headers look like one of these:

-----BEGIN OPENSSH PRIVATE KEY-----
...
-----END OPENSSH PRIVATE KEY-----
-----BEGIN RSA PRIVATE KEY-----
...
-----END RSA PRIVATE KEY-----
Do not use a public key. Files starting with ssh-rsa AAAA... or BEGIN SSH2 PUBLIC KEY are public keys and will not work with ssh -i.

Manual PuTTY Fallback

If the generated Windows ssh.exe config connects but immediately disconnects, Windows OpenSSH may not be handling MCP stdio correctly in that environment. Use PuTTY plink.exe only as a fallback.

Prerequisites

  • PuTTY installed on your Windows machine.
  • Your SSH private key file.

Step 1: Convert Your SSH Key To PuTTY Format

  1. Open PuTTYgen.
  2. Click Load and select your private key file.
  3. Click Save private key and save as a .ppk file.

Step 2: Create A PuTTY Saved Session

  1. Open PuTTY.
  2. Set Host Name to the VM IP and Port to 22.
  3. Set keepalives to 30 seconds.
  4. Set auto-login username to opc.
  5. Browse to your .ppk key file under SSH authentication settings.
  6. Save the session as logan-mcp.

Step 3: Cache The Host Key

"C:\Program Files\PuTTY\plink.exe" -load logan-mcp "echo hello"

Type y when prompted. You should see hello.

Step 4: Configure Your MCP Client

{
  "mcpServers": {
    "logan-mcp": {
      "command": "C:\\Program Files\\PuTTY\\plink.exe",
      "args": [
        "-load", "logan-mcp",
        "-batch",
        "-T",
        "cd /home/opc/logan-mcp-server && source venv/bin/activate && oci-logan-mcp --user firstname.lastname"
      ]
    }
  }
}

Troubleshooting

Symptom Likely Cause Fix
Username must be firstname.lastname The username contains digits, spaces, underscores, hyphens, or extra dots. Use letters only with exactly one dot, for example rishabh.ghosh.
Missing SSH key logan.key is not beside the installer. Put logan.key in the same folder as Double-Click-to-Install.cmd.
Access to ...\.logan-mcp\logan.key is denied An older installer copy hardened the key permissions and blocked overwrite. Use the updated installer, which writes a fresh key file instead of overwriting the old one.
Cannot find a process with the process identifier ... during Codex close A Codex process exited by itself while the installer was closing Codex. Use the updated installer, which ignores already-exited Codex processes.
Codex does not show logan-mcp Codex did not fully restart. Let the installer close Codex, or close all Codex processes from Task Manager and open Codex again.
Server connects then immediately disconnects Windows OpenSSH stdio handling or remote path/venv issue. First verify SSH manually. If the remote command works but Codex disconnects, use the PuTTY fallback.
No ED25519 host key is known An older installer passed the pinned known_hosts path incorrectly when the Windows username path contained a space. Use the updated installer, which passes SSH options as separate arguments.
Network error or timeout Firewall, VPN, security list, or IP access issue. Confirm port 22 to 130.162.53.112 is reachable from the user's machine.
